Activation analytics privacy notice

Pixel Forge uses a small set of coarse product milestones to understand whether the editor helps people create and export pixel art. These events contain no account, device, project, artwork, filename, palette, color, or persistent identifier.

What is collected

An event contains its approved milestone name and, where useful, one coarse category such as project source, drawing-tool family, save destination, or export format. Pixel Forge does not record pointer movements, interaction histories, free-form text, timestamps from the browser, cookies, local-storage identifiers, IP addresses, user agents, or geography in the analytics dataset.

Why it is collected

Aggregate counts help identify where the creative flow succeeds or stops. Because the events have no user or session identifier, they cannot provide unique-user, returning-user, or cohort conversion measurements.

Processor and transport

The browser sends events to a same-origin endpoint operated with Cloudflare Workers. As with any web request, Cloudflare receives ordinary network metadata needed to deliver and protect the request. Pixel Forge does not copy that metadata into Workers Analytics Engine. The Analytics Engine row contains only the approved event fields and Cloudflare's server-side time.

Retention and access

Cloudflare Workers Analytics Engine retains rows for its fixed three-month period. Access is restricted to the Pixel Forge owner through separate deployment and read-only analytics credentials. There is no browser-side analytics history to delete because none is stored there.

Questions and requests

Contact privacy@pixel-forge.app for questions or privacy requests. The identifier-free design may mean a particular analytics row cannot be connected to, found for, or deleted on behalf of an individual.

Scope

This notice covers Pixel Forge activation analytics. It does not expand the application's separate error-reporting service, and it is not a legal-compliance guarantee.
